## 1. 项目概述

### 1.1 项目目标

制作一个精美的暑假出游邀请 + 计划共创网页。发起方通过链接发送给受邀方，完成从"发出邀请"到"共同制定出游计划"的完整互动流程。

### 1.2 核心玩法

```
发送链接 → 她打开 → 邀请页（点击同意）→ 二次确认（有"那算了"但反悔无效）
→ 进入计划共创阶段（5个功能模块）
```

## 2. 视觉设计系统

### 2.1 风格参考

基于 `beautiful-html` 的 **Editorial Tri-Tone** 模板进行改造。

### 2.2 色彩体系

| 变量 | 色值 | 角色 |
|------|------|------|
| 主色 — 灰粉 (pink) | `#F2B6C6` | 封面背景、柔和底色 |
| 辅色 — 奶黄 (cream) | `#F2D86A` | 亮色区块、高亮元素 |
| 深色 — 勃艮第红 (burgundy) | `#7A1F35` | 深色背景、文字主色、强调元素 |

三色体系，不再引入第四色，保持杂志 editorial 般的克制与张力。

> **改造方向**: 在 editorial-tri-tone 的基础上，适当增加浪漫/甜蜜氛围——圆角更柔和、增加微动效（飘落花瓣、星星闪烁等）、emoji 作为点缀元素。

### 2.3 字体系统

| 用途 | 字体 | 说明 |
|------|------|------|
| 标题/展示 | **Bricolage Grotesque** (wght 400-800) | 表现力强的可变无衬线体 |
| 正文/诗意文字 | **Instrument Serif** (Regular + Italic) | 文学感衬线体 |
| 标签/数据/UI | **JetBrains Mono** | 技术感等宽字体 |

所有字体通过 Google Fonts 引入，无需本地安装。

### 2.4 动画与动效

| 位置 | 动效 | 描述 |
|------|------|------|
| 封面 | 花瓣飘落 | CSS 动画实现的粉色花瓣从顶部飘落 |
| 封面 | 文字渐入 | 标题逐字 fade-in + translateY |
| 按钮 | hover 脉冲 | 按钮 hover 时轻微 scale + 阴影扩散 |
| 过渡 | 页面切换 | 全屏 burgundy 遮罩快速 fade-in/out (350ms, cubic-bezier)，模拟丝滑翻页 |
| 入场 | 内容揭示 | 每页内容使用 `.reveal` 类：opacity 0→1 + translateY 30→0 (0.7s, ease-out-expo)，配合 `reveal-d1~d7` 错落延迟 |
| 入场 | 缩放弹入 | 按钮等关键 CTA 使用 `.reveal-scale`：scale 0.85→1 + opacity 0→1 |
| 小元素 | 心跳/闪烁 | 爱心 emoji 心跳动画 |
| 二次确认 | 俏皮弹入 | "那算了"结果文字弹性缩放入场 |

### 2.5 BGM

- **歌曲**: "Run Away With Me" — Carly Rae Jepsen
- **播放方式**:
  - 页面加载后显示一个精致的音乐播放按钮（pill 风格）
  - 用户点击后开始播放（浏览器自动播放策略兼容）
  - 循环播放，可暂停
  - 音频来源：使用合法的音频链接或本地音频文件
- **UI 设计**: 一个小巧的 pill 组件，显示 "♪ Run Away With Me" ，点击播放/暂停切换

## 3. 页面结构与交互流程

### 3.1 整体架构

```
┌──────────────────────────────────────────────┐
│                 Section 1                      │
│         封面「夏日出逃计划2」(纯展示)            │
│          点击任意位置 → 翻到邀请页              │
│                      ↓                        │
│                 Section 2                      │
│          邀请页「你愿意和我一起吗？」            │
│              只有一个"好呀"按钮                 │
│                      ↓ 点击好呀                │
│                 Section 3                      │
│              二次确认（反悔无效）                │
│    "当然了！" → 直接通过                        │
│    "那算了"   → "哼╭(╯^╰)╮，反悔也没用" → 通过  │
│                      ↓ 自动过渡                │
│             Section 4-8                        │
│            计划共创阶段（5个子页面）             │
│   📅 日期选择 → 🗺️ 目的地投票 → 📝 行程共创    │
│        → 📋 清单核对 → 💌 留言板                │
└──────────────────────────────────────────────┘
```

### 3.2 Section 1 — 邀请封面

**功能描述**:
- 全屏展示，dusty pink (`#F2B6C6`) 背景
- 顶部 meta 信息栏（"Summer · 2026" / "A Little Invitation"）
- **Pill 集群**: 散布浪漫关键词，每个 pill 使用 staggered reveal 动画依次入场
- **大标题**: "夏日出逃计划 2"，"2" 用 Instrument Serif 斜体对比
- **副标题**: Instrument Serif 斜体 "Run Away With Me"
- **底部呼吸提示**: "点击任意位置继续 ↓"，2.5s 循环呼吸动画
- **无按钮** — 点击页面任意位置 → 翻到邀请页
- 所有内容元素使用 `.reveal` + `reveal-d1~d6` 错落入场

**入场动画时序**:
| 元素 | delay |
|------|-------|
| Meta 信息栏 | 0.05s |
| Pill 集群 | 0.15s |
| 大标题 | 0.3s |
| 副标题 | 0.45s |
| 点击提示 | 0.75s |

### 3.3 Section 2 — 邀请页（新增）

**功能描述**:
- 全屏 burgundy (`#7A1F35`) 深色背景
- 中央超大文字，逐行 stagger 入场：
  > "你愿意"
  > "和我一起吗？"
- "我" 字用 Instrument Serif 斜体 + cream 色突出
- 副标题：Italic serif "— A summer to remember —"
- **核心交互**: 页面中央仅一个按钮 —— 粉色底 + cream 字的大号 pill 按钮"好呀 💕"
- 按钮使用 `.reveal-scale` 缩放弹入动画（0.45s delay）
- **只有一个"好呀"按钮，不可拒绝**
- 点击 → 翻到确认页

### 3.4 Section 3 — 二次确认（反悔无效）

**功能描述**:
- 点击"好呀 ❤️"后，全屏过渡到 cream (`#F2D86A`) 背景
- 中央大号询问文字渐入：
  > "真的吗？🥹"
- 下方出现**两个按钮**：
  | 按钮 | 文案 | 点击后 |
  |------|------|--------|
  | **主按钮** | "当然了！💕" | 直接过渡到 Section 3 |
  | **次按钮** | "那算了" | 弹出文字 "哼╭(╯^╰)╮，反悔也没用" → 短暂停顿 → 同样过渡到 Section 3 |
- **关键设计**:
  - "那算了"按钮视觉上比"当然了！"小一号、更低调（但不能太小，要可见）
  - "那算了"只是一个调皮的陷阱——点击后同样进入计划阶段，无法真正退出
  - 点击"那算了"后的提示文字使用俏皮弹入动画（spring easing），配一个鬼脸或吐舌 emoji
- 此阶段**不可回退**——一旦进入 Section 2，无法返回 Section 1

### 3.5 Section 4~8 — 计划共创阶段

采用分步式（step-by-step）设计，类似 deck 翻页，deck-stage 控制。每页一个功能模块。

#### 3.5.1 📅 日期选择器（Date Picker）

**目标**: 双方各自标记暑假空闲时间，找到交集。

**功能点**:
- 展示 7 月 ~ 8 月的双月历视图
- **视角不对称设计**: 当前浏览者自己的日历占 70% 面积（上方、大字），对方的日历占 30%（下方、小字）。sol 打开时 sol 的日历是主体，鼠打开时鼠的日历是主体。对方日历只读不可操作
- **区间快捷选择**: 点击起始日期 → 再点击结束日期，整个区间一键标记/取消。点同一个日期两次取消选择。不再需要一个一个点
- 提示栏实时反馈当前选择状态（"已选起始：2026-07-05 → 点击结束日期"）
- **交集可视化**: 两人都标记的日期高亮 + ❤️ 标记
- 底部显示交集天数统计
- 支持"建议日期"一键推荐（取最长连续交集段）
- 支持"清除我的标记"一键重置

**交互细节**:
- 自己的日历：burgundy/cream 标记，可点击操作
- 对方日历：只读展示，灰色缩小显示
- 交集日：pink 高亮 + ❤️
- 区间选择时：起始日期用奶油色高亮 + 阴影环，区间预览虚线边框

#### 3.5.2 🗺️ 目的地投票（Destination Voting）

**目标**: 列出候选目的地，双方投票选择。

**功能点**:
- **预设候选目的地**（3个）:
  | 目的地 | 描述 |
  |--------|------|
  | 🏙️ **江浙沪** | 小预算 💰 |
  | 🏔️ **广西/云南** | 穷游富游终不似少年游 |
  | ✨ **其他** | 她可以手动输入想去的其他地方 |
- 每个目的地以 card 形式展示
- **投票机制**: 每人有 3 票，可以投给任意目的地（可重复投给同一个）
- 实时显示得票排名
- 得票最高的目的地高亮为"🏆 首选目的地"
- **"其他"选项**: 点击后展开输入框，她可以添加自定义目的地，添加后自动成为一个新的投票 card

**交互细节**:
- Card 使用 editorial-tri-tone 的 `.s-grid .card` 风格（圆角 28px，burgundy/cream 交替）
- 投票点击有微动效（卡片短暂弹跳）
- 实时更新排名

#### 3.5.3 📝 行程共创（Itinerary Co-creation）

**目标**: 两人一起填写想做的事、想去的地方、想吃的东西。

**功能点**:
- 分为三个 Tab/分类：
  - 🎯 **想去的地方**（景点、地标、小店…）
  - 🍜 **想吃的东西**（餐厅、小吃、甜品…）
  - 🎪 **想做的事**（看日出、拍照、逛夜市…）
- **词云布局**: 每个分类下的项目以词云形式呈现——不同字号、随机微旋转角度，彩色药丸标签散布在容器中，生动有趣
- **添加方式**: 底部输入框 + 添加按钮，新词弹出动画入场（cloudPop spring easing）
- 每一项显示来源标签（用不同颜色/emoji 区分双方，不做性别标识）：
  - sol 发起方：burgundy 底色 + cream 文字
  - 鼠 受邀方：pink 底色 + burgundy 文字
- 每个词附带一个迷你身份标签（sol/鼠）

**交互细节**:
- 词云使用 flex-wrap 自适应排列，居中对齐
- hover 时词轻微放大 + 阴影浮现
- 字号大小根据内容长度动态变化（26px ~ 48px）
- 旋转角度在 -3.5° ~ +3.5° 之间微变，营造手写感

#### 3.5.4 📋 清单核对（Checklist）

**目标**: 出发前的准备事项清单，逐项核对。

**功能点**:
- 预设清单模板分类：
  - 🧳 **行李**: 换洗衣物、防晒、墨镜、充电器、牙膏牙刷、洗面奶、洗头膏、一次性内衣
  - 📄 **证件**: 身份证、学生证、车票/机票
  - 💊 **药品**: 晕车药、创可贴、花露水
  - 🎒 **其他**: 零食、雨伞
- 每项旁边有 checkbox，可勾选
- 支持添加自定义项目
- 进度条显示完成百分比
- 双方都可以操作，用不同颜色的勾标记是谁完成的

**交互细节**:
- 使用 editorial-tri-tone 的 chart/stat 风格展示进度
- 完成后显示庆祝微动效（小🎉飘过）

#### 3.5.5 💌 留言板（Message Board）

**目标**: 留给对方的小情话、期待感言。

**功能点**:
- 聊天式布局，每条留言显示：
  - 头像（用颜色圆点或 emoji 区分双方，不做性别区分）
  - 昵称
  - 时间戳
  - 内容
- 底部输入框 + 发送按钮
- 预设几条"启动留言"（发起方的开场白）
- 支持 emoji 😊

**交互细节**:
- 使用 editorial-tri-tone 的 quote/split 布局风格
- 字体用 Instrument Serif——让留言有手写信的感觉
- 双方留言用不同对齐方向或底色区分

## 4. 技术方案

### 4.1 前端

| 项目 | 选型 |
|------|------|
| 框架 | **纯 HTML/CSS/JS**（单文件，参考 editorial-tri-tone 模板结构） |
| 页面导航 | `deck-stage.js` 翻页系统，扩展为支持条件跳转 |
| 动画 | CSS Animations + 少量 JS 触发 |
| 字体 | Google Fonts（Bricolage Grotesque + Instrument Serif + JetBrains Mono） |
| BGM | HTML5 `<audio>` + 自定义播放控件 |
| 响应式 | 以 1920×1080 为基准设计（deck 风格） |

### 4.2 后端

| 项目 | 选型 |
|------|------|
| 后端框架 | **Python Flask**（轻量级，Python 生态成熟） |
| 数据库 | **SQLite**（轻量、零配置、适合个人小项目） |
| 数据持久化 | 所有计划数据保存到后端，刷新不丢失、跨设备可同步 |
| API 设计 | RESTful：`GET /api/plan/<id>` / `POST /api/plan/<id>` |
| 唯一标识 | 每个邀请生成唯一链接 ID（如 `/invite/a3f2k9`），她打开即绑定到此计划 |

**为什么选 Flask + SQLite 而不是纯前端？**
- 纯前端（localStorage）：数据只存在一个浏览器里，换设备就看不到，无法"发链接给她，她填完你也能看到"
- Flask + SQLite：后端存数据，双方打开同一个链接看到同一份数据，真正的"共同创作"。SQLite 就是一个文件，不需要装数据库服务。

### 4.3 数据模型

```
Invitation
  - id: string (unique link token, e.g. "a3f2k9")
  - created_at: datetime
  - status: enum("pending", "accepted", "planning", "done")
  - accepted_at: datetime (点击"当然了！"或"那算了"的时间)

Plan
  - invitation_id: FK
  - free_dates_sol: [date]       ← sol（发起方）空闲日期
  - free_dates_shu: [date]       ← 鼠（受邀方）空闲日期
  - destinations: [
      {name, description, votes_sol: int, votes_shu: int, is_custom: bool}
    ]
  - itinerary: [
      {category: "place"/"food"/"activity", content: str, author: "sol"/"shu"}
    ]
  - checklist: [
      {category: str, content: str, checked_sol: bool, checked_shu: bool}
    ]
  - messages: [
      {author: "sol"/"shu", content: str, time: datetime}
    ]
```

### 4.4 部署

| 项目 | 方案 |
|------|------|
| 部署平台 | **Vercel**（前端静态 + 后端 Serverless Function）或 **Railway**（Flask 直接跑） |
| 域名 | 平台自带域名（如 `summer-plan.vercel.app`），发链接给她 |
| 链接形式 | `https://<app-url>/invite/<unique-id>` |

## 6. 约束与规则

| 规则 | 描述 |
|------|------|
| **不可跳过** | Section 1 必须点击"好呀 ❤️"，不能滚动或按键跳过 |
| **不可真正拒绝** | Section 2 有"当然了！"和"那算了"两个按钮，但无论选哪个最终都会进入计划阶段。"那算了"只是一个调皮的假选项 |
| **不可回退** | Section 2 一旦进入，无法返回 Section 1 |
| **必须保存** | 所有计划阶段的数据通过 API 自动保存到后端，刷新不丢失 |
| **双方可见** | 两人的输入和选择在同一页面可见，营造"共同创作"感 |
| **无性别区分** | 两人均为女生，所有 UI 不做男女区分，用颜色/emoji 标识双方身份 |
